Agent Overview

All 6 agents — purpose, tools, and RAG usage

🧭 Router Agent
  • Classifies every chat message
  • Extracts structured parameters
  • Handles multi-patient ID extraction
  • Single LLM call per message
Trigger: every /chat request
📋 Registration Agent
  • Register new patients (CLN/LAB/PHM)
  • Update demographic fields
  • Re-embed after identity changes
  • Find or create patient cluster
Trigger: intent = register | update
🔄 Reconciliation Agent
  • Detects field conflicts across records
  • Calls LLM adjudication (Call 1)
  • Calls safety review (Call 2)
  • Notifies ALL cluster locations
Trigger: intent = reconcile
✓ Uses RAG — conflict type mapped
💊 Prescription Agent
  • Agentic tool loop (max 8 turns)
  • get_patient_record → check_interaction
  • issue_prescription OR block_prescription
  • Direct allergy name matching
  • Dedup: case-insensitive partial match
Trigger: intent = prescribe
✓ Uses RAG — drug_interaction + allergy
🗄️ Database Agent
  • Agentic tool loop (max 10 turns)
  • 14 tools: update, add, remove, delete
  • Auto-resolves cluster_id from source_ref_id
  • Re-embeds on identity field changes
Trigger: intent = db_update
🔍 Query Agent
  • Non-agentic pure DB queries
  • 7 query types: patients, prescriptions, escalations, medications, allergies, notifications, search
  • LLM formats output with RAG context
Trigger: intent = query
✓ Uses RAG — chat threshold (0.35)

Prescription Agent — Decision Flow

From "Prescribe X for Y" to issued or blocked

Prescription Agent Tool Loop (max 8 turns)
💬 "Prescribe [drug] for [patient_id]"
↓
🧭 Router → intent: prescribe
extracts: source_ref_id, drug, dosage
↓
TURN 1
🔧 get_patient_record(source_ref_id)
→ returns: medications[], allergies[], name, blood_type
↓
TURN 2
🔧 check_interaction(new_drug, existing_meds, allergies)
→ RAG: retrieve_for_prescription() — drug_interaction + allergy_mismatch
→ threshold 0.30 · severity re-ranked
↓
🔀 safe_to_prescribe?
= no interaction_guidelines found
AND no direct allergy_hit
YES ✓
NO ✗
↓
TURN 3
🔧 issue_prescription()
adds drug to medications[]
logs in prescriptions table
status: active
↓
✅ ISSUED
action: "issued"
↓
TURN 3
🔧 block_prescription()
logs blocked attempt
status: blocked
includes reason + interactions
↓
🚫 BLOCKED
action: "blocked"

Example: Prescribe Ibuprofen for Dengue Patient

TurnTool CalledKey Result
1get_patient_record("CLN-001")medications: ["paracetamol"], allergies: [], notes: "dengue fever"
2check_interaction("ibuprofen", ["paracetamol"], [])RAG retrieves LK-002 (CRITICAL: Dengue+NSAIDs). safe_to_prescribe: false
3block_prescription("CLN-001", "ibuprofen", "Dengue contraindication")Blocked. Logged in prescriptions table with status=blocked.

const DESTRUCTIVE_KEYWORDS = ["delete","remove record","drop","wipe","erase","clear all","purge"]

function isDestructive(msg: string): boolean {
  return DESTRUCTIVE_KEYWORDS.some(kw => msg.toLowerCase().includes(kw))
}

End-to-End Scenario Flow

🦟 Scenario A — Dengue + NSAIDs
  1. Patient CLN-001 has "dengue fever" in medications
  2. Chat: "Prescribe ibuprofen for CLN-001"
  3. get_patient_record → sees dengue
  4. check_interaction → RAG retrieves LK-002 (CRITICAL)
  5. block_prescription called
  6. Result: 🚫 BLOCKED
🔔 Scenario B — Cross-Location Notification
  1. CLN-001: blood_type A+ / LAB-001: blood_type B+
  2. Both in same cluster (linked by matching)
  3. Chat: "Compare CLN-001 and LAB-001"
  4. data_integrity conflict detected → RAG fetches DT-001
  5. Notifications created for CLN-001 AND LAB-001
  6. Switch roles → both bells show the alert
🤧 Scenario C — Allergy Block
  1. Chat: "Add allergy penicillin for CLN-001"
  2. Chat: "Prescribe amoxicillin for CLN-001"
  3. check_interaction: direct allergy_hit detected
  4. RAG fetches AL-001 (Penicillin protocol)
  5. Result: 🚫 BLOCKED — penicillin allergy
🗄️ Scenario D — Full DB Update
  1. "Add medication paracetamol for CLN-001" → cyan card
  2. "Remove medication paracetamol from CLN-001" → cyan card
  3. "Update CLN-001 phone to 0771234567" → blue card + re-embed
  4. "Resolve escalation for CLN-001" → escalation closed
  5. All operations confirmed in Supabase
