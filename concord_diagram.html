② The Agentic Loop — Core of Concord
🔄 Reconciliation Pipeline (triggered by "reconcile" intent)
1
Fan Out — Pull Records from All Sources

Fetches source_records from clinic, lab, and pharmacy tables in Supabase. Each source has its own prefix: CLN-xxx, LAB-xxx, PHM-xxx.

CODE only
│
2
Identity Matching — Vector Similarity via pgvector

Encodes name + DOB + NIC with all-MiniLM-L6-v2 (local, free). Runs cosine similarity in Supabase pgvector to cluster records that belong to the same physical patient across sources.

CODE only
│
3
Conflict Detection — Deterministic Rules (No LLM)

Compares fields across matched records: medication mismatches, allergy conflicts, blood type clashes, dosing contradictions. Pure code — fast, auditable, zero LLM cost.

CODE only
│
4
RAG Retrieval — Clinical Guidelines Injected as Context

Each conflict description is embedded and searched against medical_guidelines (pgvector). Top-k guidelines retrieved by cosine similarity, filtered by category and severity, injected into the LLM prompt.

RAG
│
5
LLM CALL 1 — Batch Adjudication

ALL conflicts sent in ONE prompt. Returns structured JSON: per conflict → trusted_value, action, rationale, confidence, severity. Uses Groq Llama 3.3-70B with structured output.

LLM CALL 1
│
6
Execute Actions — Write Resolutions to Supabase

Applies the adjudication decisions: updates source_records, writes adjudication rows, logs detected conflicts. All deterministic code — no LLM needed.

CODE only
│
7
LLM CALL 2 — Escalation Review

Reviews all resolved actions. Flags low-confidence items for clinician attention. Writes escalation rows to Supabase. Triggers notifications. This is the safety net.

LLM CALL 2
③ RAG Knowledge Base — Live at Every Decision Point

Knowledge Base

  • medical_guidelines table in Supabase
  • Each row: title, content, category, severity, tags, embedding vector
  • Categories: drug_interaction / allergy_mismatch / data_integrity
  • Severity: critical / high / medium / low

Retrieval Thresholds

  • Prescription check: 0.30 (precision)
  • Conflict resolution: 0.25 (wider net)
  • Chat / Q&A: 0.35 (confident only)
  • Re-ranked by severity weight (critical ×2.0)
  • Curator dedup threshold: 0.86

Where RAG Is Used

  • Prescription → interaction check before issue/block
  • Reconciliation → guideline per conflict type
  • Chat → answers grounded in guidelines
  • Registration → risk flags on new patient
  • Blocked Rx → safe alternative suggestions

⑥ Patient Search — 10 Methods

Direct DB Lookups (PostgreSQL)

  • source_ref_id exact match (CLN-001)
  • NIC number — exact and partial
  • Phone number — partial match
  • Name — case-insensitive partial (ilike)
  • Blood type (A+, B-, O+…)
  • Medications — Python scan of array field
  • Allergies — Python scan of array field
  • Date of birth — Python string match

Vector Similarity Fallback (RAG)

  • Only runs when all exact searches return nothing
  • Encodes search term with all-MiniLM-L6-v2
  • Cosine similarity against patient embeddings
  • Threshold: 0.60 — catches phonetic/spelling variants
  • e.g. "Nimal Perea" → finds "Nimal Perera"
